P5665 划分

发表于 2020-11-21 更新于 2020-12-23 Valine：

　
P5665一道毒瘤题。

简化题目:

将所给的数 [$a_1,a_2,…,a_n$] 分成 $K$ 个区间,

使得 $sum_1\le sum_2\le … \le sum_k (1\le k\le n)$。

这里的 $sum$ 是指划分的这段区间的加和。

求$(sum_1)^2+(sum_2)^2+(sum_3)^2+…+(sum_k)^2$的最小值。

样例解释也说的很明白了就不再解释了。

思路

①

先考虑一件事情:是区间分得越多结果越小还是分得越少结果越小?

显然,$(a+b)^2=a^2+2ab+b^2>a^2+b^2$

得出结论①:分得区间越多,那么结果更小(答案更优)。

②

若 $a+x \le b$,也就是我们既能将 $x$ 划分在左边,也能划分在右边,那么放在哪边满足最优解?

若加在左边,总和为$(a+x)^2+b^2=x^2+a^2+b^2+2ax$

若加在右边,总和为$a^2+(x+b)^2=x^2+a^2+b^2+2bx$

$\because a<b$

$\therefore x^2+a^2+b^2+2ax<x^2+a^2+b^2+2bx$

$\therefore (a+x)^2+b^2<a^2+(x+b)^2$

得出结论②:若两边都能放,加在左边结果更小(答案更优)。

我们定义 $sum_i$ 为: $1\sim i$ 的前缀和。

我们定义 $g_i$ 为: 现在划分到了点 $i$,我们暂时称之这个储存的最后一个划分点为 $pos$。

则如果当前的 $i$ 是一个划分点,则需要满足:

其中 $g_{pos}$ 就为以 $pos$ 为划分结尾的上一个划分点。

移一下项：$sum_i>2sum_{pos}-sum_{g_{pos}}$ 也就是说 $i$ 如果是划分点就要满足这个式子$(pos$ 是在 $i$ 之前的一个划分点$)$

因为结论②,我们发现每个 $i$ 的位置越靠后越好,所以我们可以使用单调队列优化。

依次从小到大放入,如果这个数比当前的队头大,且满足 $sum_i>2sum_{pos}-sum_{g_{pos}}$就可以清空队列。每次都将队头存入 $g_i$。

输入

接下来我们解决当 $type=1$ 的时的输入情况:

#include <cstdio>
const long long Mod = 1073741824; 
int n,type,a,b[MAXN],g[MAXN];
long long sum[MAXN];

long long p[MAXM],l[MAXM],r[MAXM];
void input()
{
    if(type==0)
    {
        for(int i=1;i<=n;i++)
        {
            scanf("%d",&a);
            sum[i]=sum[i-1]+a;
        }
        return;
    }
    long long x,y,z,m;
    scanf("%lld%lld%lld%d%d%lld",&x,&y,&z,&b[1],&b[2],&m);
    for(int i=1;i<=m;i++)
        scanf("%lld%lld%lld",&p[i],&l[i],&r[i]);
    for(int i=3;i<=n;i++)
        b[i]=((x*b[i-1]%Mod)+(y*b[i-2]%Mod)+z%Mod)%Mod;
    for(int j=1;j<=m;j++)
        for(int i=p[j-1]+1;i<=p[j];i++)
        {
            a=b[i]%(r[j]-l[j]+1)+l[j];
            sum[i]=sum[i-1]+a;
        }
}
